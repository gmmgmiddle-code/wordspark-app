# Project WordSpark Guidelines

ไฟล์นี้บรรจุข้อกำหนดและแนวทางการพัฒนาสำหรับโปรเจกต์ **WordSpark App** เพื่อให้ Gemini CLI ปฏิบัติตามอย่างเคร่งครัดในทุกครั้งที่เข้ามาทำงานในโปรเจกต์นี้

## 1. เป้าหมายและฟังก์ชันหลัก (Core Objectives)
- **แอปพลิเคชัน:** Web App สำหรับสุ่มคำศัพท์ตามหมวดหมู่เพื่อฝึกการเล่าเรื่อง (Storytelling)
- **หมวดหมู่เริ่มต้น:** สถานที่ (Places), อารมณ์ (Emotions), และสัตว์ (Animals)
- **กลไกการทำงาน:** กดสุ่ม 1 ครั้ง แสดงผล 1 คำ (One word at a time) และสามารถกดสุ่มใหม่ได้ทันที

## 2. แนวทางการพัฒนาด้านเทคนิค (Technical Standards)
- **Framework:** React 19 (Vite) โดยใช้ Functional Components และ Hooks เท่านั้น
- **Styling:** เน้นใช้ Vanilla CSS (App.css / index.css) เพื่อความเรียบง่ายและยืดหยุ่นสูง
- **Data Source:** ดึงข้อมูลคำศัพท์จากไฟล์ `.csv` ที่จะถูกระบุไว้ในโฟลเดอร์โปรเจกต์
- **Coding Style:** เขียนโค้ดที่สะอาด (Clean Code), มีคอมเมนต์อธิบายฟังก์ชันสำคัญ และเป็นไปตามมาตรฐาน React สมัยใหม่

## 3. การออกแบบและการสื่อสาร (UI/UX & Language)
- **ภาษา:** UI หลักของแอปพลิเคชันต้องเป็น **ภาษาไทย** ทั้งหมด ยกเว้นส่วนที่เป็นชื่อทางเทคนิค
- **โทนสีและสไตล์:** ต้องปฏิบัติตามข้อกำหนดในไฟล์ `referrences/web-tone.md` (ถ้ามี) เพื่อให้ดีไซน์ออกมาตรงตามความต้องการของผู้ใช้
- **ความง่ายในการใช้งาน:** เน้น UI ที่สะอาดตา (Clean UI) และการตอบสนองที่รวดเร็ว (Responsive Design)

## 4. ขั้นตอนการทำงาน (Workflow Constraints)
- **ความต่อเนื่อง:** ทุกครั้งที่เริ่มการทำงานใหม่ ให้ตรวจสอบไฟล์ `README.md`, `GEMINI.md`, และไฟล์ข้อมูลใน `referrences/` ก่อนเริ่มเสมอ
- **การยืนยัน:** หากมีการเปลี่ยนแปลงโครงสร้างแอปพลิเคชันอย่างมีนัยสำคัญ ต้องแจ้งและขอคำยืนยันจากผู้ใช้ก่อนดำเนินการ
- **ความปลอดภัย:** ห้ามบันทึกข้อมูลส่วนตัวหรือ API Key ใดๆ ลงในโค้ดหรือไฟล์ในโปรเจกต์
